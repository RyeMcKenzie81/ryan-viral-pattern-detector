## Key Interactions to Verify
- Mode toggle on Tab 1 switches between Competitor and Quick URL forms cleanly.
- Pasting a known-good FB post URL → spinner appears → pack created → redirect.
- Pasting an invalid/private FB URL → error message + upload form appears with URL preserved.
- Dragging a 50MB .mp4 file → upload completes, pack created.
- Pasting a URL that was analyzed before (same org) → modal offers "View existing" or "Re-pack now".
- Re-pack action → completes in <5s (no Gemini call), pack appears with status='complete'.
- Sidebar pack history shows quick packs with "Quick: …" label, competitor packs with competitor name.
- Pack page auto-refreshes from `pending` → `complete` once worker runs.

## Edge Cases
- 100MB+ upload → rejected with clear message (don't crash).
- Non-FB URL → rejected before resolver runs.
- Photo post URL (not video) → resolver returns ResolverError, falls back to upload.
- Private FB video URL → resolver returns ResolverError, falls back to upload.
- Audio-stripped FB video → succeeds; warning shown on pack page.
- Superuser viewing org="all" → quick mode still works, pack assigned to brand's real org.
- Two users in same org submit same URL within 5 seconds → only one pack created (unique partial index).
- Worker job fails 3 times → pack status='failed', error message visible, "Retry" button works.
- Migration applied to DB with existing competitor packs → all existing packs still load.

## Critical Paths
- **End-to-end Quick URL:** Tab 1 paste URL → resolver → upload → pack insert → job enqueue → worker analysis → Tab 2 video details → Tab 3 remix → save to angle pipeline → candidates with `source_type=QUICK_INTEL`.
- **End-to-end upload fallback:** Tab 1 bad URL → error → upload form → file selected → pack insert → worker analysis → same downstream as URL path.
- **Dup URL re-pack:** Tab 1 paste URL → modal → re-pack → instant pack with copied extraction → Tab 2/3 work.
- **Competitor regression:** Tab 1 Competitor mode → existing flow works exactly as before migration.
