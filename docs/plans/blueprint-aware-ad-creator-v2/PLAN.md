# Blueprint-Aware Ad Creator V2

## Context

Ad Creator V2 currently generates ads using template style references, product data, hooks, and offer variant context. It checks congruence against LP hero data from `brand_landing_pages`. However, users have rich **landing page blueprints** (generated by the Landing Page Analyzer's Skill 5) that contain section-by-section strategy, tone direction, copy directions, key differentiators, and emotional hooks — all tailored to a specific product and brand.

**Goal**: Let users optionally select a saved blueprint when creating V2 ads, so the generated ads are congruent with and specifically designed for the landing page the blueprint represents. This enriches the generation prompt with landing page strategy context and enhances congruence scoring with a 4th dimension.

---

## Architecture: Data Flow

```
UI (21b) → scheduled_jobs.parameters.blueprint_id
  → Worker unpacks params.get('blueprint_id')
    → run_ad_creation_v2(blueprint_id=...)
      → AdCreationPipelineState.blueprint_id
        → FetchContextNode: loads blueprint, extracts context → state.blueprint_context
          → HeadlineCongruenceNode: passes blueprint_context as 4th dimension
          → GenerateAdsNode: passes blueprint_context to prompt construction
            → ReviewAdsNode: saves blueprint_id on each generated_ad row
```

### 4. FetchContextNode: `viraltracker/pipelines/ad_creation_v2/nodes/fetch_context.py`

After the LP hero data fetch block (after line 279), add a new non-fatal block.

**Key defensive checks** (from QA review):
- Use `rb.get("sections") or []` (not just `rb.get("sections", [])`) to handle `sections: null`
- Guard `sections[0]` access behind `if sections`
- Handle both nested `{reconstruction_blueprint: {...}}` and root-level JSON structures

```python
# Blueprint context (optional, non-fatal)
if ctx.state.blueprint_id:
    try:
        from viraltracker.services.landing_page_analysis.blueprint_service import (
            ReconstructionBlueprintService,
        )
        bp_svc = ReconstructionBlueprintService()
        bp_row = bp_svc.get_blueprint(ctx.state.blueprint_id)

        if bp_row and bp_row.get("status") == "completed":
            bp_json = bp_row.get("blueprint") or {}
            rb = bp_json.get("reconstruction_blueprint", bp_json)
            strategy = rb.get("strategy_summary") or {}
            sections = rb.get("sections") or []  # Defensive: handles null
            populated = [s for s in sections if s.get("content_status") == "populated"]

            ctx.state.blueprint_context = {
                "blueprint_id": ctx.state.blueprint_id,
                "source_url": bp_row.get("source_url"),
                "strategy_tone": strategy.get("tone_direction"),
                "key_differentiators": (strategy.get("key_differentiators") or [])[:5],
                "awareness_adaptation": strategy.get("awareness_adaptation"),
                "architecture_recommendation": strategy.get("architecture_recommendation"),
                "hero_copy_direction": sections[0].get("copy_direction") if sections else None,
                "hero_emotional_hook": (sections[0].get("brand_mapping") or {}).get("emotional_hook") if sections else None,
                "top_copy_directions": [
                    s["copy_direction"] for s in sections[:5]
                    if s.get("copy_direction")
                ],
                "sections_covered": len(populated),
                "total_sections": len(sections),
            }

            # Enrich lp_hero_data from blueprint if not already loaded
            if not ctx.state.lp_hero_data and sections:
                hero = sections[0]
                hero_mapping = hero.get("brand_mapping") or {}
                ctx.state.lp_hero_data = {
                    "hero_headline": hero_mapping.get("primary_content", ""),
                    "hero_subheadline": hero_mapping.get("emotional_hook", ""),
                    "key_claims": strategy.get("key_differentiators") or [],
                }
                logger.info("Blueprint enriched lp_hero_data (no existing LP data)")

            logger.info(
                f"Blueprint context loaded: {len(populated)}/{len(sections)} sections populated"
            )
        else:
            logger.warning(f"Blueprint {ctx.state.blueprint_id} not found or not completed")
    except Exception as e:
        logger.warning(f"Failed to load blueprint context (non-fatal): {e}")
```

## Edge Cases Handled

| Scenario | Behavior |
|----------|----------|
| No blueprint selected | All new fields are None/optional — zero behavior change |
| Blueprint is "partial" or "failed" | FetchContextNode checks `status == "completed"`, skips otherwise |
| Blueprint for different product | UI filters by product_id + session state clearing on product change |
| Blueprint has no sections | `sections or []` defensive check; `blueprint_context` has `sections_covered=0` |
| Blueprint `sections` is null | `rb.get("sections") or []` handles both missing and null |
| Blueprint JSON has no `reconstruction_blueprint` key | `bp_json.get("reconstruction_blueprint", bp_json)` falls back to root |
| LP hero data already exists | Blueprint does NOT overwrite — only fills in if `lp_hero_data` is None |
| Congruence LLM fails | Non-fatal fallback to 1.0 score (existing pattern preserved) |
| Old jobs without blueprint_id | `params.get('blueprint_id')` returns None — zero impact |
| Blueprint deleted after job submitted | FK `ON DELETE SET NULL` — generated_ads keep NULL; FetchContextNode skips gracefully |
| User changes product after selecting blueprint | Session state guard clears `v2_blueprint_id` when product changes |
| 4th congruence dimension lowers scores | Simple average: each dim now 0.25 instead of 0.33 — acceptable since blueprint_alignment typically scores well for aligned content |

## QA Findings Addressed

From parallel QA agent review:

- **DefectScanNode + RetryRejectedNode**: Both have `save_generated_ad()` calls that MUST include `blueprint_id` (3 total save call sites beyond ReviewAdsNode) -> Added as explicit steps 12-13
- **Session state race condition**: User could change product but keep stale blueprint_id -> Added session state guard in UI
- **Null sections defense**: `rb.get("sections", [])` doesn't protect against `sections: null` -> Changed to `rb.get("sections") or []`
- **Blueprint JSON nesting variance**: Some blueprints nest under `reconstruction_blueprint`, others don't -> Already handled by `bp_json.get("reconstruction_blueprint", bp_json)` fallback
- **Congruence weighting**: 4th dimension reduces per-dimension weight from 0.33 to 0.25 -> Acceptable; simple average is the existing pattern and blueprint_alignment scores well for aligned content
- **No upstream/downstream breaks**: `generated_ads` queries use `SELECT *` or explicit columns that don't include `blueprint_id` -> nullable column addition is safe
- **State serialization**: `to_dict()` uses generic `dataclasses.fields()` -> new fields auto-serialize, no special handling needed
